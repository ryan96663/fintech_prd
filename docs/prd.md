# “保e家” APP 产品需求文档

# 一，概述

此产品为依托中国工商银行的保险集成平台。

## 产品介绍

“保 e 家”是一个针对家庭的互联网保险集成平台，基于用户/家庭成员信息、经济状况、预期保障目标、家庭风险构成等信息生成完全契合用户利益和需求的保险保障组合。

## 产品定位

- 量身定制家庭的保险配置，以合适的保费满足家庭保障的核心需求，还可避免冗余
- 集成多家保险公司产品，提供充足信息并方便产品对比
- 保险自选品到报销的全流程一体化服务平台
- 自个体到家庭的保险方案制定，避免合成谬误

## 产品设计思考

- 信息不对称是保险业的一大问题，客户对自己资产的实际价值、家庭经济现状、成员健康情况拥有更多信息，另一方面保险公司对保险产品的理赔条件、实际利率、相关条款了解更加深刻（虽这些信息会在保险合同当中著名，但是由于客户普遍未能拥有较高的法律知识和金融知识导致对条款的理解没有那么深刻）。
- 此外，家庭对自身想达到的保障有认知但认知模糊，且对风险敞口认知也存在缺口。
- 且针对家庭的保险，无论是寿险还是财险都属于标准化金融产品（非针对个人量身定制，而是由公司统一制定），当家庭需要在多个平台购买保险产品服务时，可能造成保障覆盖重复或保费投入冗余，当家庭选择在一个平台购买保险产品时想要达到最终目标\-\-“成本较低（保费投入较低）”，“收益较高（切实契合资深的风险/满足自己的保障预期）”具有一定困难。
- 与传统的互联网保险平台或纯线下代理人模式相比，“保e家”的核心区别在于
    - 基于AI与大数据驱动分析技术，摒弃传统盲目推销
    - 基于用户真实信息综合衡量经济与风险敞口，先进行保障分析，再进行产品配置
    - 打破信息壁垒
    - 跨产品类别归一化
    - 综合全部保险产品的全生命周期服务闭环

# 二，需求分析

## 需求背景

- 信息不对称问题破解——人找保险→真实的需求找保险
    一方面，用户最了解自身的资产真实价值、家庭经济现状及成员健康状况；另一方面，保险公司掌握着理赔条件、实际费率及复杂的免责条款。由于普通家庭缺乏专业的金融与法律知识，可能对冗长的保险条款了解不全面、理解不深刻。
    保司方信息不对称可能导致保险公司销售员不能全方位、综合视角地提供保险产品，造成风险遗漏；从客户角度来看，可能因条款了解不全面导致“理赔难问题”。

- 家庭保障视角的需求定位于风险敞口识别
    大部分家庭具有“买保险防范风险”的初步意识，但对家庭真实的“风险敞口”在哪里、到底需要多大额度的保障缺乏科学、量化的认知，往往只能凭直觉或盲从他人购买。

- 产品配置到保险赔付的全生命周期服务
    传统保险交易易出现“重销售、轻服务”问题，对保险报销部分的服务投入较低；
    跨平台管理保单对客户而言存在统筹成本，甚至导致保单失效（脱保）；
    贯穿售后的保险理赔通道与协助推进服务，能有效打破行业的信任危机，同时建立用户粘性。

- 标准化产品与个性化需求错配
    市面上的寿险、财险多为保险公司统一制定的标准化金融产品。当用户在多个平台分散购买时，极易造成保障范围重复覆盖或保费投入冗余。
    而在单一平台购买，又很难跨保司挑选到真正契合自身风险预期且“成本较低、收益较高（保障杠杆最高）”的产品组合。

## 市场背景

1）市场规模与趋势\-\-稳步增长，相较于国际水平仍有成长空间

随着我国经济快速发展，保险密度在过去的7年内总体呈稳步增加态势，7年间增幅达到48%；受疫情影响，2020\-2022年保险深度下降，2022年后逐渐回暖，至2024年恢复到4\.2%。

从长期发展维度看，我国保险深度与国际平均水平仍存在显著差距，行业存在较大提升空间，我国作为发展中国家，居民基础性支出压力大，长期保障需求层次有待提升。

2）保险消费者行为分析：由于网络发展，消费者信息获取渠道拓宽，保险的传统代理人销售模式正转向为从网络信息处获取保险信息并购买的方式。购买渠道存在转移到线上购买的趋势，购买原因更多为风险意识的提升和家庭/自身健康状况的改变。

①购买渠道：2025年消费者在保产品的购买渠道当中，传统保险公司线上渠道首度超越其线下渠道，成为消费者购险的第一入口。

②购买因素：主要归因为个人风险意识的提升，成员/个人身体状况的变化，家庭责任提升。（购买原因主要还是人找保险，而不是真实需求找保险）

③信息获取渠道：向周围人了解是消费者了解保险的首选方式，其次是通过社交媒体和短视频渠 道；通过AI工具了解保险占比达到16%，实现了大幅增长。

④消费者痛点：消费者在了解保险时最偏好产品对比/测评、理赔案例/指南、知识科普等实用且真实的信 息，并偏好案例故事、短视频、图文文章等内容形式；**认为当前内容存在晦涩难懂、真假难辨、销售 导向过强等不足**

3）目标用户：20\-40岁年轻群体、中高收入者、高等教育人群

互联网保险购买正在从线上转线下：消费者从线下转到线上的TOP3原因分别是投保方便、缴费方式灵活、信息更透 明；分险种来看，信息透明是互联网储蓄型保险/寿险的最显著优势，缴费方式灵活是互联网重疾 险的最突出特征

目标用户定位：消费者收入越高，越倾向在线上渠道购险；细分渠道来看，20\-30岁的年轻群体偏好在 互联网保险中介、互联网保险公司购险，50岁以上的中老年群体倾向在传统保险公司线上渠道购险

### 5\.1 全局说明

#### 5\.1\.1 登录/未登录

|**页面**|**未登录态**|**登录拦截方式**|
|---|---|---|
|首页|可浏览产品和科普；功能入口点击时拦截|弹出登录模态|
|发现|可搜索、浏览、对比；购买操作拦截|弹出登录模态|
|家庭|Tab不做隐藏，点击直接要求登录|弹出登录模态|
|我的|Tab不做隐藏，点击直接要求登录|弹出登录模态|

- **T：** 未登录用户点击需登录态的功能入口或"家庭""我的"Ta
- **R：** 登录模态全屏弹出，不可下滑关闭，必须完成登录或返回
- **F：** 模态底部弹出 300ms；登录成功 toast 2s 后自动进入目标功能
- **E：** 登录接口超时→提示重试，不关闭模态

#### 5\.1\.2 导航模式

- **T：** 用户点击底部Tab / 页面内入口
- **R：** 底部Tab栏4项常驻（首页\|发现\|家庭\|我的）；二级页push右侧滑入，顶部返回按钮回退；键盘弹起时Tab隐藏
- **F：** Tab切换即时；push动画 250ms
- **E：** 连续点击返回→防抖≥300ms

#### 5\.1\.3 全局组件

**AI客服悬浮按钮：**

- **T：** 所有页面右下角常驻，用户点击唤起
- **R：** 半屏对话窗（60%高度），可拖至全屏；支持文字\+语音；覆盖方案解读/产品咨询/理赔指导/保单查询/操作指引/投诉场景
- **F：** 对话窗弹出 250ms；AI流式打字机回复；语音输入波纹动画；未读消息红点角标
- **E：** AI超时→回复引导转人工；语音识别失败→提示重试或打字；人工排队\>10人→提示等待时长

**消息通知（首页右上角）：**

- **T：** 点击消息图标 → 进入消息中心
- **R：** 消息分两类：系统消息（可删）、保单消息（强制保留不可删）；时间倒序
- **F：** 未读红点角标（\>99显示"99\+"）；保单消息左侧橙色标记
- **E：** 列表加载失败→空状态\+下拉刷新

**页面结构：** 顶部栏 → 摘要数字面板 → 功能入口区 → 热门产品推荐

- **R：** 左侧App名称，右侧消息图标；首页不设设置入口

- **T：** 已登录用户进入首页
- **R：** 5项数字卡片横排可滑动：待缴费 / 保单临期 / 理赔中 / 保障缺口 / 草稿与购物车；点击跳对应详情

### 5\.7 关键流程

#### 5\.7\.1 评估→方案→购买主流程

首页\[风险评估\] → 问卷作答 → 需求定位\(loading\) → 评估报告页 → \[生成方案\] → loading → 方案推荐页 → \[微调/确认\] → 订单确认 → 核保 → 支付 → 保单生成

- 每一步均自动保存草稿，防止数据丢失
- 支付为唯一不可逆节点，支付前所有步骤可回退修改

#### 5\.7\.2 理赔主流程

\[发起理赔\] → 选择保单 → 出险登记 → AI判断理赔类型 → 材料清单\+上传 → AI材料核验 → 提交报案 → 进度追踪 → 赔付到账
- 材料上传后AI核验，缺漏实时提示补正
- 进度关键节点强提醒推送

#### 5\.7\.3 注册与绑定流程

注册\(手机号\+验证码\+密码\) → 协议确认 → 实名认证\(身份证OCR\+真人验证\) → 工行账户绑定 → 信息完善引导\(可跳过\)
